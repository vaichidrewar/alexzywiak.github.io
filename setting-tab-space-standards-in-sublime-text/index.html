Setting Tab/Space Standards in Sublime Text

09 October 2015

An analogy for you; Properly indented code is to mixed spaces and tabs as Oxford's English Dictionary is to a horde of tasmanian devils in a swimming pool full of scrabble letters.

Yeah, properly indenting code is important. It may look pretty in your editor, but on git or elsewhere out in the real world, it could look like a tangled mess if you're not consistent.

I'm going to cover how to change the default tab settings in Sublime.

To get going, navigate over to:
Sublime Text > Preferences > Settings - User.

Settings in Sublime are controlled using JSON notation.

While playing around in there, you can modify your tab settings with the following commands.

"tab_size":Number
Set it equal to the number of spaces per tab.
Default value is 4.

"translateTabsToSpaces":true/false
If true, all tab keypresses will be recorded as the number of spaces specified in "tab_size" instead of a tab character. Visually, the tab key will function the same, but there will be no tab characters whatsoever.
Default value is false.

"useTabStops":true/false
If true, will set tab and backspace to add/delete up to the closest tab stop regardless of whether it's a tab character or a series of spaces.
Defaults to true.

Here is an example of setting all tabs to be recorded as two spaces as per HR style guide.

{
  "tab_size":2,
  "translateTabsToSpaces":true,
  "useTabStops":true
}
